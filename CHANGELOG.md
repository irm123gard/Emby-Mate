- `MAJOR`：不兼容变更（接口语义、配置方式、关键行为断裂）。

## [2.4.9] - 2026-03-23

> 本版内容主要整理自 `2026-03-20` 之后这一轮主线稳定性、运行时性能和后台交互性能优化，优先展示对实际播放体验和日常维护最有感知的变化。

### Highlights

- **主线播放稳定性继续优先**：首播媒体请求遇到 `followed/html 4xx` 时会更保守地回退，避免把 HTML 错页直接交给播放器；同时补上短时 `head window / jump window` 和早期 jump 的窄范围预算放宽，重点改善“刚点播放”和“第一次拖动时间轴”这两段最脆弱的体验。
- **排障能力更直接**：后台新增可持久化的 `debugProxyHeaders` 开关，仅在管理员主动开启时，才会给异常媒体请求附加 `X-Proxy-*` 诊断头，方便直接通过浏览器 Network 或 HAR 判断 `route / final host / status / type / playback mode / upstream cache`。
- **运行时与后台性能继续收紧**：节点列表与 CF 指标读取改为优先走轻量节点索引，metadata prewarm 收紧触发条件与并发，播放窗口预算统计去掉整表排序，后台列表、搜索、`tcping`、RTT、节点活跃和 CF 指标自动刷新都补上了去重、短窗口复用和隐藏页跳过策略。

### Added

- 新增高级设置项 `debugProxyHeaders`，默认关闭，用于按需开启代理诊断头。
- 新增短时播放窗口能力：`head window`、`jump window`、会话级 early jump assist 和对应运行时状态。
- 新增运行时 inflight 清理覆盖范围，开始回收陈旧的外部重定向解析状态和 metadata prewarm 状态。
- 新增 `worker-v2.4.9.js` 作为 `v2.4.9` 的发布快照与 Dashboard 粘贴版。

### Changed

- 启播主链路新增 `followed/html 4xx` 保守回退逻辑，优先减少“点击播放后立即退出播放窗口”这类失败体验。
- 节点列表、CF 指标和站点路径统计优先读取 `sys:nodes-index` 轻量索引，降低 `KV list + N 次 get` 的 fan-out 压力。
- `metadata prewarm` 仅对更高价值的 JSON 响应触发，并增加大小门槛、并发限制和 inflight 去重，减少和真实媒体请求抢预算。
- 播放窗口预算统计改为增量字节记账与更轻的淘汰逻辑，不再依赖整表排序选 victim。
- 后台节点列表改为增量复用渲染，CF 卡片和节点活跃改为变化感知刷新，搜索、`tcping`、RTT、节点活跃和 CF 指标自动刷新都加入短时结果复用与隐藏页跳过。
- 运行时缓存分步清理从“过期项懒删”扩展为“过期项 + 超量项 + stale inflight”组合回收，进一步压低长时间运行后的状态膨胀。

### Fixed

- 修复 `v2.4.7` 导出的旧配置在 `v2.4.8+` 导入时，历史 `secret -> path` 迁移缺口导致站点路径丢失的问题。
- 修复移动端输入法场景下，“设置 / CF 设置 / 第三方反代”最后一次编辑值没有在点击保存时正确提交到 KV 的问题。
- 修复节点列表在节点较多时额外读取最近访问缓存，导致后台 `action:list` 触发 Worker subrequest 上限、前端误显示“暂无节点”的问题。
- 修复移动端菜单主题切换、暗色模式输入框背景、CF 指标卡窄布局以及多类后台细节刷新中的视觉与交互不一致问题。

### Docs

- README 升级到 `v2.4.9`，同步更新发布快照文件名、发布重点和当前发布仓库的部署说明。
- 补充 `v2.4.9` 发布记录，集中整理这轮播放稳定性、性能和后台体验优化。

## [2.4.8] - 2026-03-20

> 本版内容主要整理自 `2026-03-16` 至 `2026-03-20` 的主线改动，优先展示对日常使用最有感知的优化点。

### Highlights

- **后台体验更统一，移动端更顺手**：管理后台按 Kumo-inspired 思路继续做组件化整理，表单、按钮、卡片、分段切换与弹窗交互统一收口；暗色模式输入框、移动端主题切换图标和弹窗底部操作区体验一起修顺。
- **CF 指标更聚焦真实站点问题**：指标卡固定为“近24小时 + 当前域名下已配置站点路径”，同时把 `Top10` 路径的服务端过滤和缓存签名一起收紧，减少无关流量噪音，更适合直接排查哪个站点在吃流量。
- **设置维护成本继续下降**：第三方反代默认改为 `暂无记录`，由管理员手动添加；节点路径短语映射、默认重定向词库拆为独立文件维护，后续改词库和预设不再需要翻主脚本。
- **发布与仓库准备更完整**：版本切到 `v2.4.8`，README / AGENTS / 发布脚本 / 分析脚本同步收口，GitHub 发布目录可直接导出并推送。

### Added

- 新增 `worker-v2.4.8.js` 作为 `v2.4.8` 的发布快照与 Dashboard 粘贴版。
- 新增独立的节点路径短语预设文件 `src/admin/ui/node-path-phrase-map.js`，方便长期维护常见中文站点名到 slug 的映射。
- 新增独立的默认重定向词库文件 `src/config/default-redirect-rule-terms.js`，将基础预设从默认配置逻辑中解耦。

### Changed

- 管理后台源码继续按 `admin / app / storage / proxy` 分层整理，保持“源码模块化 + 单文件产物部署”的主线不变。
- 管理后台样式按 Kumo-inspired 思路继续收敛为 `token + primitive + variant` 结构，统一按钮、面板、输入框、分段切换、pill 和移动端 sheet 弹窗体验。
- `CF指标` 卡片固定为近24小时口径，聚焦“当前域名下已配置站点路径”的站点流量、请求数与 `Top10` 路径，界面和权限说明同步简化。
- `Top10` 路径查询改为在 Cloudflare 查询阶段就按已配置站点路径过滤，避免 `admin`、探测接口或其他杂路径挤占榜单。
- `activity` / 最近使用相关缓存改为跟随当前节点路径签名失效，节点集合变化后能更快拿到正确活动数据。
- 第三方反代默认改为空记录状态，不再预填历史示例，和重定向白名单的维护方式保持一致。
- 移除节点路径单字拼音 `CHAR_MAP` 预设，改为只保留短语级映射，避免中文输入被过度猜测成不稳定的单字拼音。
- `analyze:deadcode` 的发布快照忽略规则改为 `worker-v*.js`，避免后续每次发版都手工补脚本名单。
- 模块分析脚本改为基于当前仓库目录运行，不再写死本地绝对路径，便于直接推送到 GitHub 后复用。

### Fixed

- 修复移动端菜单中的主题切换图标在暗色模式下不同步的问题。
- 修复暗色模式下输入框背景与 segmented 控件样式不统一的问题。
- 修复 CF 指标卡“站点流量”字段在窄布局下过早换成多行的问题。
- 修复 `Top10` 路径统计可能混入未配置站点路径的问题，减少卡片展示与实际业务入口不一致的情况。
- 修复节点活动缓存未随节点集合变化同步失效的问题，降低最近使用/活动数据滞后的概率。
- 清理 `admin-script`、`settings-model`、`cf-analytics` 等模块中因功能演进遗留的死代码与历史 helper。
- 登录兼容请求头中的版本标识同步更新到 `v2.4.8`，避免快照与运行时版本口径错位。

### Docs

- README 升级到 `v2.4.8`，同步发布入口、近24小时 CF 指标口径、GitHub 发布目录和当前源码结构说明。
- AGENTS 更新为当前 `src/` 目录结构与模块职责，不再保留旧 `src/core`、`src/ui`、`src/proxy/proxy.js` 口径。
- `cloudflare-console-checklist.md` 升级到当前主线版本，并修正旧仓库绝对路径引用。

## [2.4.7] - 2026-03-13

### Added

- 新增播放序列级回归样例与 HAR 夹具，覆盖 `Forward`、`EplayerX`、`Lenna`、`SenPlayer`、`CapyPlayer` 等常见链路场景。
- 新增 GitHub 最小发布目录导出流程：`npm run release:github` 会生成 `.publish/github`，仅保留发布所需 4 个文件。
- 新增动态版本语法校验脚本 `scripts/check-release.mjs`，发布检查不再依赖硬编码版本号。
- 新增 `worker-v2.4.7.js` 作为 `v2.4.7` 的发布快照与 Dashboard 粘贴版。

### Changed

- 外部重定向缓存继续收敛为“规范化 key + 等价 URL 复用 + 并发去重 + GET/HEAD 共用”的统一语义，减少同媒体重复解析与重复回源。
- `PlaybackInfo` 阶段预解析改为更保守策略，配合预算保护闸在高压场景自动降级到轻量路由，优先保障可播放性。
- 播放窗口优化链路补齐：首播窗口、尾部小 Range 探测、拖动落点窗口之间明确优先级，避免互相抢占导致的卡顿。
- `release-snapshot`、`bootstrap-src`、`check` 全部改为读取 `package.json` 版本，发布流程从手工改版本切换为脚本化。

### Fixed

- 修复多播放器下 `DirectStreamUrl` 重复拼接导致的异常路径（如 `/embyemby/`、`/emby/https://...`、重复 basepath）引发的 `404`。
- 修复部分链路下同 URL 多次重定向解析未复用结果导致的起播等待偏长问题。
- 修复高频小包 Range 与频繁重连场景下的缓存命中不稳定问题，降低“可播但长时间等待”的体感概率。
- 修复发布流程中只改版本号却遗漏脚本同步的风险点，避免发布快照文件名与校验脚本不一致。

### Docs

- README 升级到 `v2.4.7`，同步发布方式、版本入口、GitHub 最小发布目录与一键部署说明。
- 更新变更记录口径，按“2026-03-11 之后的优化工作”集中整理本版发布内容。

## [2.4.6] - 2026-03-10

### Added

- 节点配置正式支持有序 `targets[]` 多目标源站，后台可按主源/备用源顺序手动排序并导入导出保序。
- 播放链路新增保守型多源回切：仅对 `GET/HEAD` 首个上游请求阶段启用，支持在连接失败或 `5xx/522-530` 时切到后续备用源。
- 代理诊断头新增目标观测字段：`X-Proxy-Target-Host`、`X-Proxy-Target-Index`、`X-Proxy-Target-Count`、`X-Proxy-Failover`、`X-Proxy-Failover-Reason`。
- 管理台新增全局 `客户端 RTT` 指标，显示当前浏览器到 Cloudflare 边缘的往返时延，支持颜色分级与手动刷新。
- 设置弹窗新增可维护的重定向白名单列表，支持 `名称 + 域名` 结构化维护，最多 10 条记录。
- 新增 `worker-v2.4.6.js` 作为 `v2.4.6` 的发布快照与 Dashboard 粘贴版。

### Changed

- 节点数据模型升级为以 `targets` 为真实语义，旧 `target` 继续保留为兼容字段并始终映射到第一顺位主源。
- TCP/HEAD 延迟探测、第三方反代地址复制和卡片展示统一以第一顺位主源为准。
- `CF设置` 从通用设置弹窗中拆分为右上角独立入口，Cloudflare 指标与 DNS 管理配置单独维护。
- 重定向白名单管理从右上角菜单迁移到“设置 -> 第三方反代”下方，交互结构与第三方反代保持一致。
- `PlaybackInfo` URL 策略细分为 `absolute / relative-bare / relative-leading-slash`，并按播放器类型分别应用到 `Forward`、`EplayerX`、`Lenna` 等客户端。
- 外部重定向缓存链路增强为“规范化缓存键 + 并发去重 + 等价媒体 URL 复用 + GET/HEAD 共用 + PlaybackInfo 保守预解析”。
- 图片缓存诊断头 `X-Emby-Proxy-Cache` 改为反映真实 `CF-Cache-Status`，带 `tag` 的图片改为更长边缘缓存。
- 构建与发布脚本同步切换到 `v2.4.6`，`check` 现会先生成发布快照再做语法校验。

### Fixed

- 修复外部重定向跟随时仍可能携带 `Origin`、`Referer` 或 Emby 凭据的问题，避免跨域回源泄露敏感头。
- 修复部分 Emby 登录链路只认 `Authorization` 或只认 `X-Emby-Authorization` 时的兼容性问题，并将补头范围收敛到 `Users/AuthenticateByName`。
- 修复多目标节点在运行时切到备用源后，`Location`、播放信息和流媒体文本改写仍沿用主源 host 的错配问题。
- 修复播放响应对已知长度媒体内容被错误改成 `chunked` 的兼容问题，保留 `206 + Content-Length` 语义。
- 修复普通 API 响应被误套用固定长度流包装，导致登录和管理接口返回空响应的问题。
- 修复 `Forward`、`EplayerX`、`Lenna` 等播放器对 `DirectStreamUrl` 重复拼接前缀、绝对 URL 嵌套、`/embyemby/`、`/embyvideos/` 等路径异常导致的 404。
- 修复管理后台内联脚本在目标源站表单和换行字符串场景下的转义错误，避免页面空白或初始化失败。
- 修复重定向白名单与设置弹窗整理后的一系列文案、布局、空状态和移动端交互细节问题。

### Docs

- README 同步更新到 `v2.4.6`，补充本版发布重点、播放器兼容策略、客户端 RTT、设置结构调整与部署入口说明。

## [2.4.4] - 2026-03-09

### Added

- CF 指标时间范围扩展为：近 3 天、近 1 周、近 1 月。
- 首页 CF 指标新增当前 host 的 `Top10` 路径视图，按第一层路径分组、合并请求数与流量，并在卡片内自动滚动展示。
- 播放代理增加路径诊断头：`X-Proxy-Path-In`、`X-Proxy-Path-Normalized`、`X-Proxy-Upstream-Path`，便于定位播放器拼接异常。

### Changed

- CF 指标卡重构为以当前 host 为中心，主视图调整为 `总流量 + 请求数 + Top10`。
- `总流量` 改为优先使用 Cloudflare Zone 级 host 流量查询，不再以 Worker 总流量作为主展示口径。
- 播放链接 canonical 化策略增强：对 `EplayerX`、`Forward`、`Lenna` 等会二次拼接路径的客户端，优先返回更合适的 `relative-node` 形式，并保留 Worker 侧异常路径归一化兜底。
- Top 路径展示策略调整为：按流量第一排序、请求数第二排序，只显示第一层路径分组。

### Fixed

- 修复长时间范围（近 3 天 / 近 1 周 / 近 1 月）下，CF 指标因无条件触发 Worker 长范围查询而整卡无数据的问题。
- 修复 `Config is not defined` 前端报错，站点“最近使用”自动刷新改为使用显式注入的前端常量。
- 修复当前 host 流量、请求数、Top 路径的权限说明不准确问题，补齐 `区域 - 分析 - 读取（Zone Analytics Read）` 与资源范围要求。
- 修复多种播放器把绝对 `DirectStreamUrl` 再次拼接后导致的重复前缀、404 或非规范播放地址问题。

### Docs

- 更新设置页与 README 中的 Cloudflare Token 权限说明，区分 `CF 指标` 与 `优选域名/IP` 两类权限。

## [2.4.1] - 2026-03-06

### Added

- 代理响应新增诊断头：`Server-Timing`、`X-Proxy-Route`、`X-Proxy-Redirect-Hops`、`X-CF-Placement`（有值时透传）。
- `cfMetrics` 接口新增可选字段：`cacheHit`、`generatedAt`（向后兼容）。
- 新增发布文档：`CHANGELOG.md`，并在 README 增加版本管理说明。

### Changed

- 外域重定向跟随升级为安全多跳（最多 3 跳），含环路检测与错误诊断。
- 流媒体改写增加体积保护阈值：PlaybackInfo JSON（512KB）、文本改写（1MB）。
- 流媒体响应缓存策略改为：`Cache-Control: no-store, no-transform`。
- Tcping 从“总是并发 itdog + socket”改为“先 itdog，失败再 socket fallback”，并增加 in-flight 去重。
- CF 指标查询增加短 TTL 缓存与并发去重，降低重复 GraphQL 压力。
- 管理页卡片交互优化为局部渲染（显隐、菜单、Tcping 刷新），搜索仅在结果变化时重绘。
- Worker 主文件重命名：`worker-v2.4-UI改版.js` → `worker-v2.4.js`，并同步 `wrangler.jsonc`。

### Fixed

- 降低大响应文本/JSON改写导致的 CPU 与内存抖动风险。
- 降低手动刷新与自动轮询并发时的 CF 指标重复请求问题。

### Docs

- README 重构为发布版文档：功能特点、部署、配置、权限、排障、版本建议。
